# 海事文档生成系统

## 简介

这是一个用于生成海事靠泊申请和进出港安全保障方案的Web应用程序。系统支持湛江港不同泊位类型的文档自动生成，大大提高了海事文档的编写效率。

## 功能特点

- 支持7种泊位类型：200、210、300、400、601泊位及400移泊300、601移泊300
- 自动生成海事局申请文档和进出港安全保障方案
- 历史记录管理功能
- 一键下载生成的Word文档
- 现代化的Web界面，操作简便

## 系统要求

- Python 3.7+
- 现代Web浏览器（Chrome、Firefox、Edge等）

## 安装步骤

1. **安装依赖包**
   ```bash
   pip3 install -r requirements.txt
   ```

2. **确认模板文件**
   - 确保 `/Users/air/Desktop/海事申请和进出港保障方案模版` 目录存在
   - 所有模板文件应为 `.docx` 格式

3. **启动应用**
   ```bash
   python3 app.py
   ```

4. **访问应用**
   - 在浏览器中打开：`http://localhost:8000`

## 使用说明

### 填写船舶信息

1. **基本信息**
   - 中文船名：例如：粤电8
   - 英文船名：例如：YUE DIAN 8
   - 代理公司：例如：中国湛江外轮代理有限公司

2. **时间信息**
   - 到港时间：船舶预计抵达湛江港的时间
   - 靠泊时间：计划靠泊码头的时间

3. **船舶参数**
   - 卸货量：计划卸货的数量（吨）
   - 整船货量：船舶总载货量（吨）
   - 船长：船舶总长度（米）
   - 船宽：船舶最大宽度（米）
   - 载重吨：船舶载重吨位（DWT）
   - 总吨：船舶总吨位（GT）
   - 净吨：船舶净吨位（NT）

4. **泊位信息**
   - 泊位类型：根据实际情况选择对应的泊位类型

### 生成文档

点击"生成文档"按钮，系统将自动：
1. 根据选择的泊位类型加载对应的模板
2. 替换模板中的示例数据
3. 生成海事局申请文档
4. 生成进出港安全保障方案
5. 保存到历史记录

### 下载文档

在历史记录中，可以：
- 点击"申请文档"下载海事局申请文件
- 点击"保障方案"下载进出港安全保障方案
- 点击"删除"移除该记录

## 项目结构

```
maritime-doc-generator/
├── app.py                      # 主应用文件
├── requirements.txt            # 依赖包列表
├── README.md                   # 说明文档
├── static/                     # 静态文件目录
│   └── index.html             # 前端页面
├── output/                     # 生成的文档输出目录
├── maritime_documents.db       # SQLite数据库文件
└── 海事申请和进出港保障方案模版/  # 模板文件目录
```

## 技术栈

- **后端**：FastAPI
- **前端**：原生HTML + CSS + JavaScript
- **数据库**：SQLite
- **文档处理**：python-docx
- **模板引擎**：Jinja2

## 注意事项

1. 确保模板文件格式正确，所有文件应为 `.docx` 格式
2. 生成的文档保存在 `output` 目录中
3. 数据库文件 `maritime_documents.db` 包含所有历史记录
4. 建议定期备份输出目录和数据库文件

## 故障排除

### 问题：无法启动应用
- 检查是否安装了所有依赖包
- 确认Python版本为3.7或以上
- 检查8000端口是否被占用

### 问题：文档生成失败
- 确认模板文件存在且格式正确
- 检查输入数据是否完整
- 查看控制台错误信息

### 问题：无法下载文档
- 确认文档已成功生成
- 检查浏览器下载设置
- 查看控制台错误信息
